# ScholarFlow 桌面应用使用（保姆级教程）

本教程说明如何 **下载安装** ScholarFlow **桌面版**（Windows / macOS）、**首次配置 API Key**、通过 **拖放 PDF** 使用，并说明 **输出文件夹** 里常见内容的含义。

> **说明**：安装包版本与下载地址以 **GitHub Releases** 为准；若某版本尚未发布桌面端，请以 README 或 Release 页说明为准。

---

## 1. 下载安装包

### 1.1 官方发布页

打开 ScholarFlow 的 GitHub 发布页：

**https://github.com/bcefghj/scholarflow/releases**

在 **Assets** 中查找适合你系统的文件：

| 系统 | 常见文件类型 | 说明 |
|------|----------------|------|
| **macOS** | `.dmg` | 磁盘映像，拖拽安装到「应用程序」 |
| **Windows** | `.exe` | 安装向导，或便携版（以 Release 说明为准） |

> **提示**：Apple Silicon（M1/M2/M3）与 Intel Mac 有时对应不同包名，请下载与芯片架构一致的版本。

### 1.2 下载注意事项

1. 优先选择 **Latest**（最新）正式版本。  
2. 若浏览器拦截下载，选择「保留」或「仍要下载」。  
3. 校验文件名与 SHA256（若 Release 提供校验和），防止文件损坏。

---

## 2. 安装步骤

### 2.1 macOS（.dmg）

1. 双击打开 `.dmg` 文件。  
2. 将 **ScholarFlow** 图标拖入 **应用程序（Applications）** 文件夹。  
3. 弹出磁盘映像，将 `.dmg` 推出（可选）。  
4. 在「启动台」或「应用程序」中首次打开 ScholarFlow。

若系统提示「无法验证开发者」：

1. 打开 **系统设置 → 隐私与安全性**。  
2. 在「已阻止使用 ScholarFlow」处点击 **仍要打开**；或右键应用图标选择 **打开** 一次以加入例外。

### 2.2 Windows（.exe）

1. 双击安装包，若出现 **用户账户控制（UAC）**，选择 **是**。  
2. 按向导选择安装路径（可使用默认路径）。  
3. 可选：勾选 **创建桌面快捷方式**。  
4. 完成安装后，从开始菜单或桌面快捷方式启动 ScholarFlow。

> **警告**：请只从 **官方 GitHub Releases** 或项目文档指明的渠道下载，避免第三方修改版携带恶意软件。

## 3. 首次启动与 API Key 配置

桌面应用与 CLI 一样，需要 **大模型 API** 才能分析论文。首次打开时通常会提示配置密钥。

### 3.1 准备密钥

在打开应用前，建议先准备好至少一种密钥（申请步骤见 **01-环境准备.md**）：

- **OpenAI**：`OPENAI_API_KEY`（`sk-` 开头）  
- **DeepSeek**：`DEEPSEEK_API_KEY`  
- **MiniMax**：`MINIMAX_API_KEY`  

### 3.2 在应用内填写

1. 打开 ScholarFlow 桌面版。  
2. 找到 **设置 / Settings** 或首次引导页中的 **API Key** 输入框。  
3. 粘贴密钥，选择或确认 **默认模型**（若应用提供下拉选项）。  
4. 保存后，部分版本会提供 **测试连接** 按钮，建议先测试再处理大文件。

> **安全提示**：密钥仅保存在本机（具体存储方式以应用实现为准），勿与他人共享屏幕时暴露。

### 3.3 若应用支持读取环境变量

部分桌面应用会继承系统环境变量。你也可以在系统环境变量中预先设置 `OPENAI_API_KEY` 等，再启动应用（具体是否支持以应用说明为准）。

## 4. 拖放 PDF 使用

### 4.1 基本操作

1. 准备一篇 **论文 PDF**（建议文字版 PDF，扫描版可能影响识别）。  
2. 打开 ScholarFlow 桌面应用。  
3. 将 PDF **拖入** 窗口中的 **拖放区域**（或按界面提示的「选择文件」）。  
4. 勾选需要的输出类型，例如：**摘要、PPT、笔记、思维导图** 等（以当前版本界面为准）。  
5. 点击 **开始 / 生成** 并等待进度完成。

### 4.2 使用建议

1. **首次使用**：先用一篇体积较小（如 5～20 页）的 PDF 测试流程。  
2. **网络**：需要调用云端 API，请保持网络畅通。  
3. **耗时**：全文生成可能比「仅摘要」更久，请耐心等待。  
4. **失败重试**：若超时，可缩小任务范围（仅生成摘要或 PPT）后重试。

> **提示**：若界面支持 **标题 / arXiv / DOI** 输入，也可不拖文件，与命令行 `sf full --arxiv ...` 类似（以桌面版功能为准）。

## 5. 输出文件夹说明

生成完成后，应用通常会提供 **打开输出目录** 按钮，或默认保存到用户文档下的某个文件夹。不同版本路径可能略有差异，以下为 **常见输出类型** 说明，便于对照整理：

| 类型 | 常见格式 | 用途简述 |
|------|----------|----------|
| 简要介绍 / 摘要 | `.md` | 快速了解论文内容与是否值得精读 |
| 汇报 PPT | `.pptx` | 组会演示，可用 PowerPoint / WPS 编辑 |
| Beamer | `.pdf` | LaTeX 幻灯片，学术风格 |
| 讲解稿 | `.pdf` | 配合幻灯片逐页讲解 |
| 学习笔记 | `.pdf` | deep / exam / quick 等模式 |
| 思维导图 | `.html` | 浏览器打开交互查看 |
| 学术海报 | `.html` | 浏览器打开预览或打印 |
| 双语翻译 | `.md` | 中英对照摘要 |

### 5.1 如何查找输出目录

1. 在应用内点击 **打开文件夹 / 在资源管理器中显示**（若有）。  
2. 或在设置中查看 **默认输出路径**。  
3. macOS 可用 **访达** 搜索文件名；Windows 可用 **资源管理器** 搜索。

### 5.2 整理建议

1. 按 **论文题目 / 日期** 建立子文件夹，避免混在同一目录。  
2. 重要结果可同步到网盘或 Git 私有仓库（**勿提交含 API Key 的配置**）。

## 6. 常见问题

| 问题 | 可能原因 | 处理建议 |
|------|----------|----------|
| 安装包无法打开 | 系统安全策略 | 见上文 macOS「无法验证开发者」、Windows「智能筛选」说明 |
| 生成失败 | API Key 无效、欠费、网络 | 检查密钥与厂商控制台余额 |
| 无 LaTeX PDF | 本机未装 Tectonic | 桌面版若依赖本地编译，参考 **01-环境准备.md** 安装 Tectonic；或仅使用不依赖 LaTeX 的输出 |
| 拖入 PDF 无反应 | 格式不支持或文件损坏 | 换一篇 PDF 或重新导出 PDF |

## 7. 与命令行 / Web 的关系

- **命令行**：适合脚本与批量（见 **02-本地安装使用.md**）。  
- **Web / Docker**：适合团队共享（见 **03-服务器部署.md**）。  
- **桌面版**：适合日常本地拖拽、可视化操作。

根据习惯选择即可，核心能力均依赖 **API Key** 与 **论文输入**。
