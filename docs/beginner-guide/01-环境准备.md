# ScholarFlow 环境准备（保姆级教程）

本教程面向**完全零基础**用户，手把手说明：安装 Python、使用 pip、申请大模型 API Key，以及安装 Tectonic（LaTeX 编译器），为后续使用 ScholarFlow 打好基础。

---

## 1. 安装 Python（Windows / macOS）

ScholarFlow 需要 **Python 3.9 及以上**版本。下面分别说明 Windows 与 Mac 的安装方式。

### 1.1 Windows

1. 打开浏览器，访问 Python 官方下载页：  
   **https://www.python.org/downloads/windows/**
2. 选择 **Latest Python 3 Release**（或页面推荐的 Windows installer）。
3. 下载 **Windows installer (64-bit)**（绝大多数电脑选 64 位即可）。
4. 运行安装程序，**务必勾选**界面底部的 **「Add python.exe to PATH」**（将 Python 加入环境变量），再点击 **Install Now**。
5. 安装完成后，按 `Win + R`，输入 `cmd` 回车，在命令提示符中执行：

   ```bash
   python --version
   ```

   若显示类似 `Python 3.12.x`，说明安装成功。

> **提示**：若提示「不是内部或外部命令」，请关闭并重新打开终端，或检查安装时是否勾选了加入 PATH。

### 1.2 macOS

**方式 A：官网安装包（适合新手）**

1. 打开：**https://www.python.org/downloads/macos/**
2. 下载 **macOS 64-bit universal2 installer**（或页面标注适用于 Apple Silicon / Intel 的安装包）。
3. 双击 `.pkg` 按向导完成安装。
4. 打开 **终端**（Spotlight 搜索「终端」），执行：

   ```bash
   python3 --version
   ```

   看到 `Python 3.x.x` 即成功。

**方式 B：Homebrew（熟悉命令行的用户）**

```bash
brew install python
python3 --version
```

> **说明**：在 Mac 上系统可能自带较旧的 `python3`，建议以 `python3` 为准；下文若写 `python`，在 Mac 上请自行换成 `python3`，或按官方文档配置 `python` 别名。

## 2. 如何使用 pip

`pip` 是 Python 的包管理器，用来安装 ScholarFlow 等第三方库。

### 2.1 检查 pip 是否可用

在终端（Windows 用 **命令提示符** 或 **PowerShell**）执行：

```bash
python -m pip --version
```

在 macOS 上若只有 `python3`：

```bash
python3 -m pip --version
```

能显示版本号即可。

> **推荐写法**：始终使用 `python -m pip`（或 `python3 -m pip`），可避免「装到了别的 Python 里」的常见问题。

### 2.2 常用命令

| 作用 | 命令示例 |
|------|----------|
| 升级 pip 自身 | `python -m pip install --upgrade pip` |
| 安装包 | `python -m pip install 包名` |
| 卸载包 | `python -m pip uninstall 包名` |
| 查看已安装列表 | `python -m pip list` |

国内网络较慢时，可临时使用镜像（任选其一）：

```bash
python -m pip install -i https://pypi.tuna.tsinghua.edu.cn/simple 包名
```

## 3. 申请 API Key（OpenAI、DeepSeek、MiniMax）

ScholarFlow 通过大模型 API 分析论文，你至少需要配置 **其中一种** 服务的密钥。下面分平台说明。

### 3.1 OpenAI

1. 打开：**https://platform.openai.com/**
2. 注册 / 登录账号（可能需要海外手机号或支付方式，以官网当前要求为准）。
3. 进入 **API keys** 页面：  
   **https://platform.openai.com/api-keys**
4. 点击 **Create new secret key**，复制生成的密钥（通常以 `sk-` 开头）。
5. **妥善保存**：密钥只显示一次，丢失需重新创建。

> **安全提示**：不要把 Key 发到公开群聊、GitHub 或截图外泄。ScholarFlow 支持用环境变量或本地配置文件保存，勿提交到代码仓库。

### 3.2 DeepSeek

1. 打开 DeepSeek 开放平台：**https://platform.deepseek.com/**
2. 注册并登录。
3. 在控制台中找到 **API Keys**（或「密钥管理」类入口）。
4. 创建新密钥并复制保存。

DeepSeek 通常对国内用户较友好，可作为 OpenAI 的替代之一。

### 3.3 MiniMax

1. 打开 MiniMax 开放平台：**https://platform.minimaxi.com/**
2. 完成注册与登录（按页面提示完成实名/企业认证等，以平台要求为准）。
3. 进入 **用户中心 / 接口密钥**（文档中常称「创建 API Key」）：  
   可参考文档中心：**https://platform.minimaxi.com/docs/**
4. 创建密钥后**立即复制**；部分平台规则下密钥仅展示一次。

> **提示**：ScholarFlow 的 `.env.example` 中使用变量名 `MINIMAX_API_KEY`。若使用 MiniMax 的 M 系列模型，请在 ScholarFlow 中按文档设置对应模型名（如 `minimax/MiniMax-M2.7`）。

### 3.4 在 ScholarFlow 里怎么用这些 Key？

后续安装 ScholarFlow 后，可以用其中任一方式（教程 **02-本地安装使用** 会细讲）：

- 环境变量：`OPENAI_API_KEY`、`DEEPSEEK_API_KEY`、`MINIMAX_API_KEY` 等；
- 命令：`sf config set api-key sk-xxx`（会自动按前缀识别厂商）。

## 4. 安装 Tectonic（LaTeX 编译器，用于生成 PDF）

ScholarFlow 会生成 **Beamer 幻灯片、讲解稿、学习笔记** 等 LaTeX 内容，需要编译为 PDF。项目**优先使用 Tectonic**（体积小、依赖少）；若未安装 Tectonic，会尝试 `pdflatex` / `xelatex`（需完整 TeX Live / MiKTeX）。

### 4.1 为什么选 Tectonic？

- 不需要安装整套 TeX Live，对新手更友好；
- ScholarFlow 源码中推荐的安装方式之一即为 Tectonic。

### 4.2 macOS 安装 Tectonic

若已安装 [Homebrew](https://brew.sh/)：

```bash
brew install tectonic
tectonic --version
```

### 4.2 Windows 安装 Tectonic

任选一种方式：

**方式 A：Scoop（推荐，简单）**

1. 按 Scoop 官网说明安装 Scoop：**https://scoop.sh/**
2. 执行：

   ```powershell
   scoop install tectonic
   tectonic --version
   ```

**方式 B：从 GitHub Release 下载**

1. 打开 Tectonic 发布页：**https://github.com/tectonic-typesetting/tectonic/releases**
2. 下载适合 Windows 的压缩包（名称中通常含 `windows` 或 `x86_64-pc-windows`）。
3. 解压后将 `tectonic.exe` 所在目录加入系统 **PATH**，新开终端后执行 `tectonic --version` 验证。

**方式 C：已安装 Rust 的用户**

```bash
cargo install tectonic
```

### 4.3 验证是否安装成功

在终端执行：

```bash
tectonic --version
```

若显示版本号，即表示 ScholarFlow 在需要编译 LaTeX 时可以调用 Tectonic。

> **注意**：若暂不安装任何 LaTeX 编译器，部分需要 PDF 的功能会报错；PPT（.pptx）、HTML 等不依赖 Tectonic 的功能仍可尝试使用。
