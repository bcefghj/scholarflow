# ScholarFlow 本地安装与使用（保姆级教程）

本教程说明如何在个人电脑上 **用 pip 安装 ScholarFlow**、**配置 API Key**、运行 **`sf scan` / `sf full` / `sf slides` / `sf notes`** 等常用命令，并整理 **常见问题排查**。

> **前置条件**：已完成 Python 与 pip 环境，并至少持有一种大模型 API Key。若尚未完成，请先阅读 **01-环境准备.md**。

---

## 1. 安装 ScholarFlow

在终端（Windows：**命令提示符 / PowerShell**；macOS：**终端**）中执行：

```bash
python -m pip install scholarflow
```

若你使用 `python3`：

```bash
python3 -m pip install scholarflow
```

国内网络较慢时，可加上清华镜像示例：

```bash
python -m pip install -i https://pypi.tuna.tsinghua.edu.cn/simple scholarflow
```

### 1.1 验证安装

```bash
sf --help
```

若能看到子命令列表（如 `full`、`scan`、`slides` 等），说明 CLI 已就绪。

> **提示**：`sf` 是 ScholarFlow 提供的命令行入口（在 `pyproject.toml` 中注册）。若提示找不到 `sf`，请确认 **安装时使用的 Python** 与 **当前终端** 为同一个（例如都用 `python3`），并检查该 Python 的 **Scripts**（Windows）或 **bin**（macOS/Linux）目录是否已加入 **PATH**。也可尝试：`python -m pip show -f scholarflow` 查看安装文件位置。

## 2. 配置 API Key

ScholarFlow 需要调用大模型，你必须配置 **至少一个** 厂商的密钥。常用方式有三种。

### 2.1 方式一：环境变量（适合临时测试）

**Linux / macOS：**

```bash
export OPENAI_API_KEY="sk-你的密钥"
```

**Windows（CMD）：**

```cmd
set OPENAI_API_KEY=sk-你的密钥
```

**Windows（PowerShell）：**

```powershell
$env:OPENAI_API_KEY="sk-你的密钥"
```

其他厂商示例：

```bash
export DEEPSEEK_API_KEY="你的DeepSeek密钥"
export MINIMAX_API_KEY="你的MiniMax密钥"
```

### 2.2 方式二：`sf config`（推荐，写入本地配置）

```bash
sf config set api-key sk-你的密钥
```

程序会根据密钥前缀等因素识别厂商。设置后可查看：

```bash
sf config show
```

### 2.3 方式三：`.env` 文件（适合与项目或脚本一起管理）

若你从源码目录运行或自行维护目录，可复制项目里的 `.env.example` 为 `.env`，按说明填写：

```env
OPENAI_API_KEY=sk-xxx
DEEPSEEK_API_KEY=
MINIMAX_API_KEY=
```

并设置默认模型等（详见 `.env.example` 注释）。

### 2.4 指定模型（可选）

例如使用 MiniMax：

```bash
sf config set model minimax/MiniMax-M2.7
```

使用 DeepSeek：

```bash
sf config set model deepseek/deepseek-chat
```

具体模型名以 [LiteLLM 与厂商文档](https://docs.litellm.ai/) 为准。

> **警告**：切勿将含真实 Key 的 `.env` 或截图提交到 Git 公开仓库。

## 3. 常用命令示例

以下示例假定你已有一张论文 PDF，文件名为 `paper.pdf`，且与当前终端工作目录一致。

### 3.1 `sf scan` — 快速筛选「值不值得读」

适合先快速看摘要与推荐指数，耗时相对短：

```bash
sf scan paper.pdf
```

不提供 PDF 时，可用标题 / arXiv / DOI（与 `sf full` 类似）：

```bash
sf scan --arxiv 1706.03762
sf scan --title "Attention Is All You Need"
```

### 3.2 `sf full` — 一键生成全套材料

生成摘要、PPT、笔记、思维导图等多种输出（具体以当前版本为准）：

```bash
sf full paper.pdf
```

指定中文输出：

```bash
sf full paper.pdf --lang zh
```

仅用 arXiv 编号（自动下载）：

```bash
sf full --arxiv 1706.03762 --lang zh
```

指定输出目录：

```bash
sf full paper.pdf --output ./my_output
```

### 3.3 `sf slides` — 只生成演示文稿相关

生成 PowerPoint：

```bash
sf slides paper.pdf
```

指定 PPT 主题（示例）：

```bash
sf slides paper.pdf --theme academic_blue
```

若需要 **Beamer PDF**，可加 `--beamer`（需本机 LaTeX/Tectonic，见环境准备篇）：

```bash
sf slides paper.pdf --beamer
```

### 3.4 `sf notes` — 学习笔记

笔记有多种模式，例如 **考试复习**：

```bash
sf notes paper.pdf --mode exam
```

**深度学习**、**快速笔记**：

```bash
sf notes paper.pdf --mode deep
sf notes paper.pdf --mode quick
```

### 3.5 其他常用命令（一览）

| 命令 | 作用 |
|------|------|
| `sf script paper.pdf` | 生成讲解稿（LaTeX PDF 等） |
| `sf mindmap paper.pdf` | 生成思维导图（HTML） |
| `sf poster paper.pdf` | 生成学术海报（HTML） |
| `sf translate paper.pdf` | 双语摘要 |
| `sf batch ./papers/` | 批量处理文件夹内 PDF |

### 3.6 通用参数提示

- `--title` / `-t`：按标题自动检索下载  
- `--arxiv` / `-a`：arXiv ID  
- `--doi` / `-d`：DOI  
- `--model` / `-m`：指定模型  
- `--lang` / `-l`：`zh` 或 `en`  
- `--output` / `-o`：输出目录  

完整列表见项目根目录 **README.md** 中的「命令参考」。

## 4. 常见问题与排查

### 4.1 提示 API Key 无效或未设置

1. 确认已执行 `export` 或 `sf config set api-key`，且当前终端会话未关闭（环境变量仅对当前窗口生效，除非写入 shell 配置文件）。  
2. 确认密钥复制完整，无多余空格或换行。  
3. 若使用非 OpenAI 厂商，检查是否设置了 `DEEPSEEK_API_KEY` / `MINIMAX_API_KEY`，且 `sf config set model` 指向该厂商支持的模型名。

### 4.2 提示找不到 LaTeX / tectonic

生成 **Beamer、讲解稿、部分笔记 PDF** 时需要编译器。请按 **01-环境准备.md** 安装 **Tectonic** 或完整 TeX Live / MiKTeX。  
若暂时不需要 PDF，可先用 **`sf slides`（不加 `--beamer`）** 生成 `.pptx`。

### 4.3 `pip install` 很慢或超时

使用国内 PyPI 镜像（见上文 `-i https://pypi.tuna.tsinghua.edu.cn/simple`），或更换网络时段重试。

### 4.4 下载 arXiv 论文失败

检查网络是否能访问 arXiv；必要时使用代理或稍后重试。也可本地下好 PDF 后使用 `sf full paper.pdf`。

### 4.5 Windows 上 `sf` 不是内部命令

1. 确认安装 ScholarFlow 时使用的 Python 与当前终端一致。  
2. 尝试 `python -m pip show scholarflow` 查看安装位置。  
3. 将 Python 的 `Scripts` 目录加入 PATH（安装 Python 时勾选「Add to PATH」可避免多数问题）。

### 4.6 PDF 无法解析或页数过大

确保文件为合法 PDF；过大或扫描版可能影响效果。可尝试换一篇论文测试是否为个别文件问题。

---

## 5. 小结

- 使用 **`pip install scholarflow`** 安装，**`sf`** 作为统一入口。  
- **API Key** 可用环境变量、`sf config` 或 `.env` 管理。  
- **`sf scan`** 快速筛选，**`sf full`** 全套生成，**`sf slides` / `sf notes`** 按需单独生成。  
- 遇错时先查 **Key、网络、LaTeX 是否安装** 这三类原因。
