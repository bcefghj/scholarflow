# ScholarFlow 服务器部署（保姆级教程）

本教程说明如何在 **云服务器** 上部署 ScholarFlow 的 **Docker 版本**（含 Nginx 反向代理占位），并概述 **域名绑定**、**HTTPS（Let’s Encrypt）** 与 **防火墙** 配置思路。

> **说明**：以下命令以 **Linux（Ubuntu 22.04 等常见发行版）** 为例。实际界面以各云厂商控制台为准。

---

## 1. 注册并购买云服务器

### 1.1 常见厂商（示例）

| 类型 | 说明 |
|------|------|
| **腾讯云** | 国内访问稳定，有学生机 / 新用户活动（以官网为准）：https://cloud.tencent.com/ |
| **阿里云** | 同样有轻量应用服务器与学生优惠（以官网为准）：https://www.aliyun.com/ |
| **DigitalOcean** | 海外节点，按小时计费，适合海外网络场景：https://www.digitalocean.com/ |

### 1.2 选购建议（新手）

1. **系统**：选择 **Ubuntu 22.04 LTS** 或 **Debian 12**（文档多、兼容性好）。  
2. **配置**：ScholarFlow Web + Docker 建议 **≥ 2 vCPU、≥ 4GB 内存**（论文解析与 LLM 调用较吃资源，可按预算上调）。  
3. **带宽**：建议 **≥ 5 Mbps**，上传大 PDF 时体验更好。  
4. 记录：**公网 IP**、**root 或初始用户名密码**（或 SSH 密钥）。

> **提示**：国内服务器部署面向国内用户时，通常需完成 **域名备案**（若使用 80/443 与国内机房），具体以法规与云厂商要求为准。

## 2. SSH 连接服务器

### 2.1 在本地打开终端

**macOS / Linux**：自带终端即可。  
**Windows**：可用 **PowerShell**、**Windows Terminal**，或安装 [PuTTY](https://www.putty.org/)。

### 2.2 使用密码登录（示例）

```bash
ssh root@你的服务器公网IP
```

首次连接会提示确认主机指纹，输入 `yes`，再输入密码。

### 2.3 使用密钥登录（推荐）

```bash
ssh -i ~/.ssh/你的私钥.pem ubuntu@你的服务器公网IP
```

云厂商控制台通常会提供下载 `.pem` 或生成密钥对的说明。

> **安全提示**：勿向他人泄露私钥；可在服务器上禁用 root 密码登录，仅允许密钥。

## 3. 安装 Docker 与 Docker Compose

### 3.1 卸载旧版本（如有，可选）

按 Docker 官方文档操作即可，此处从略。

### 3.2 使用官方便捷脚本（Ubuntu 示例）

请参考 Docker 官网最新步骤：**https://docs.docker.com/engine/install/ubuntu/**

常见流程概要：

```bash
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### 3.3 验证

```bash
sudo docker run hello-world
docker compose version
```

将当前用户加入 `docker` 组（可选，避免每次 `sudo`）：

```bash
sudo usermod -aG docker $USER
```

重新登录后生效。

## 4. 使用 Docker Compose 部署 ScholarFlow

项目仓库中已提供 **`docker/docker-compose.yml`**，默认会构建应用镜像并启动 **Nginx**。

### 4.1 克隆代码

```bash
sudo apt-get install -y git
git clone https://github.com/bcefghj/scholarflow.git
cd scholarflow
```

### 4.2 配置环境变量

```bash
cp .env.example .env
nano .env
```

至少填写一种 **`OPENAI_API_KEY`** / **`DEEPSEEK_API_KEY`** / **`MINIMAX_API_KEY`** 等，并可根据需要修改 `SF_MODEL`、`SF_LANG` 等。

保存退出（nano：`Ctrl+O` 回车，`Ctrl+X`）。

### 4.3 启动服务

compose 文件在 `docker` 子目录，**构建上下文为上一级目录**，请从仓库根目录执行：

```bash
docker compose -f docker/docker-compose.yml up -d
```

### 4.4 访问 Web

默认映射：

- **ScholarFlow 应用**：容器内 `8080`，经 Nginx 反代后，浏览器访问 **`http://服务器公网IP`**（Nginx 监听 `80`）。

若无法访问，检查：

1. 云厂商安全组是否放行 **80**（及 **443** 若启用 HTTPS）。  
2. 容器是否运行：`docker compose -f docker/docker-compose.yml ps`。  
3. 本机防火墙（见下文第 7 节）。

> **说明**：`docker-compose.yml` 中 **nginx** 将 HTTP 流量转发到 `scholarflow:8080`，并预留 **Let’s Encrypt** 验证目录挂载，便于申请证书。

## 5. 绑定域名

1. 在域名注册商处购买域名（腾讯云 DNSPod、阿里云万网、Cloudflare 等）。  
2. 添加 **A 记录**：主机记录常用 `@`（根域名）或 `www`，记录值填 **服务器公网 IP**。  
3. 等待 DNS 生效（通常数分钟到数小时），可用以下命令检查：

   ```bash
   ping 你的域名.com
   nslookup 你的域名.com
   ```

4. 将 Nginx 配置中的 `server_name` 从占位改为你的域名（见下一节）。

## 6. 使用 Let’s Encrypt 配置 HTTPS

ScholarFlow 自带的 **`docker/nginx.conf`** 当前为 **HTTP** 监听，并在 `/.well-known/acme-challenge/` 提供 ACME 验证路径；`docker-compose` 已挂载 `certbot` 相关目录，便于你接入证书。

### 6.1 思路概要

1. 确保 **域名已解析** 到本机，且 **80 端口** 可从公网访问。  
2. 使用 **certbot**（standalone 或 webroot 模式）申请证书；证书文件可放在 compose 已挂载的 `./docker/certbot/conf` 等路径（与 `nginx.conf`、compose 卷保持一致）。  
3. 在 Nginx 中增加 **`listen 443 ssl`** 的 `server` 块，指定 `ssl_certificate` 与 `ssl_certificate_key`，并将 HTTP 重定向到 HTTPS。

### 6.2 参考命令（需在服务器上按实际路径调整）

在宿主机安装 certbot（示例）：

```bash
sudo apt-get install -y certbot
```

使用 webroot 方式（与 nginx 中 `root /var/www/certbot` 对应）时，需保证 Nginx 已能正确提供验证文件。也可先用 **standalone** 临时占用 80 端口申请（申请时需暂停 Nginx 或改用其他策略）。

证书续期可加入 **cron**：

```bash
sudo certbot renew --dry-run
```

> **警告**：HTTPS 配置错误会导致浏览器无法访问；修改前建议备份 `nginx.conf`，并在测试环境先验证。

> **提示**：若使用 **Cloudflare** 等 CDN，需注意 SSL 模式（Full / Strict）与源站证书匹配。

## 7. 防火墙与安全组设置

### 7.1 云厂商安全组（必做）

在控制台为实例绑定安全组，放行：

| 端口 | 用途 |
|------|------|
| **22** | SSH（建议仅允许你的办公 / 家庭 IP） |
| **80** | HTTP / ACME 验证 |
| **443** | HTTPS |

不建议对全网长期开放 **22**，可改为密钥 + 固定 IP 限制。

### 7.2 服务器本机防火墙（ufw 示例，Ubuntu）

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status
```

若 SSH 端口不是 22，请放行对应端口后再 `enable`，避免被锁在门外。
