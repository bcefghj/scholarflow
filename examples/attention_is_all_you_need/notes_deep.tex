\section*{学习笔记: Attention Is All You Need}

\subsection*{1. 研究背景与问题定义}

\subsubsection*{1.1 领域背景：序列转导模型的发展}

在自然语言处理和序列转导任务（如机器翻译）领域，\textbf{循环神经网络（RNN）}及其变体，包括\textbf{长短期记忆网络（LSTM）}和\textbf{门控循环单元（GRU）}，长期以来占据着主导地位。这些模型采用\textbf{编码器-解码器（Encoder-Decoder）}架构：

\begin{itemize}
  \item \textbf{编码器}：将输入序列 $(x_1, ..., x_n)$ 映射到连续表示 $z = (z_1, ..., z_n)$
  \item \textbf{解码器}：基于 $z$ ，逐个生成输出序列 $(y_1, ..., y_m)$
\end{itemize}

\subsubsection*{1.2 之前存在的核心问题}

\paragraph{1.2.1 顺序计算的固有瓶颈}

传统RNN模型的计算沿着输入和输出序列的\textbf{符号位置}因式分解：

\[
h_t = f(h_{t-1}, x_t)
\]

其中 $h_t$ 是位置 $t$ 的隐藏状态，依赖于前一隐藏状态 $h_{t-1}$ 和当前位置的输入 $x_t$。这种\textbf{固有的顺序性质}导致了两个严重问题：

\begin{enumerate}
  \item \textbf{并行化受限}：在训练样本内部无法进行并行计算
  \item \textbf{内存限制}：当序列较长时，跨样本的批处理（batching）受到限制
\end{enumerate}

虽然后续工作通过\textbf{因子分解技巧}和\textbf{条件计算}改善了计算效率，但\textbf{顺序计算的基本约束仍未解决}。

\paragraph{1.2.2 长距离依赖的学习困难}

在RNN中，两个位置之间的路径长度与它们之间的距离成正比 $O(n)$。这使得模型在\textbf{学习长距离依赖关系}时变得困难，因为梯度在反向传播过程中可能会\textbf{指数级衰减或爆炸}。

\paragraph{1.2.3 卷积方法的局限性}

为解决RNN的并行化问题，一些工作（如\textbf{ByteNet}、\textbf{ConvS2S}）使用\textbf{卷积神经网络（CNN）}作为基础构建块。然而：

\begin{itemize}
  \item \textbf{ConvS2S}：关联两个任意位置所需的运算数随距离\textbf{线性增长}
  \item \textbf{ByteNet}：随距离\textbf{对数增长}
\end{itemize}

尽管优于RNN，但仍未能达到\textbf{常数时间复杂度}的理想目标。

\subsubsection*{1.3 本文要解决的核心问题}

本文（Transformer）旨在提出一个\textbf{完全基于注意力机制}的模型架构，彻底\textbf{摒弃循环和卷积操作}，从而：

\begin{enumerate}
  \item \textbf{实现最大程度的并行化}
  \item \textbf{缩短任意位置间的依赖路径长度}至常数级
  \item \textbf{显著减少训练时间}
  \item \textbf{在翻译质量上达到或超越现有最佳模型}
\end{enumerate}

\subsubsection*{1.4 问题的重要性}

\begin{itemize}
  \item 机器翻译是NLP领域的核心任务，性能提升具有广泛的应用价值
  \item 序列转导是许多NLP任务的基础范式
  \item 训练效率的提升使得更大规模模型的训练成为可能
  \item 该架构的通用性使其可扩展到其他模态（音频、视频等）
\end{itemize}

\subsubsection*{2.2 注意力机制的演进}

\textbf{注意力机制}允许建模输入或输出序列中不同位置之间的依赖关系，而不受距离限制。在此之前，注意力机制通常\textbf{与RNN结合使用}：

\begin{itemize}
  \item \textbf{Bahdanau注意力}：加性注意力，通过前馈网络计算兼容性
  \item \textbf{Luong注意力}：乘性（点积）注意力，计算更高效
\end{itemize}

\subsubsection*{2.5 自注意力的先驱工作}

\textbf{自注意力（Self-Attention）}，也称为\textbf{内部注意力}，是一种关联同一序列中不同位置以计算序列表示的注意力机制。之前的工作已将自注意力用于：

\begin{itemize}
  \item 阅读理解
  \item 抽象摘要生成
  \item 文本蕴含
  \item 学习任务无关的句子表示
  \item 端到端记忆网络
\end{itemize}

然而，据本文所述，\textbf{Transformer是第一个完全依赖自注意力、不使用任何序列对齐RNN或卷积的转导模型}。

\subsection*{3. 核心方法详解}

\subsubsection*{3.1 整体架构概述}

\begin{figure}[htbp]
\centering
\includegraphics[width=0.8\textwidth]{fig1_p3.png}
\end{figure}

如论文中的\textbf{图1}所示，Transformer采用经典的\textbf{编码器-解码器结构}，但完全使用\textbf{堆叠的自注意力}和\textbf{逐位置的全连接层}。

\paragraph{3.1.1 编码器（Encoder）}

\begin{itemize}
  \item 由 \textbf{N = 6} 个相同层的堆叠组成
  \item 每层包含两个子层：
\end{itemize}
\begin{enumerate}
  \item \textbf{多头自注意力机制}（Multi-Head Self-Attention）
  \item \textbf{逐位置前馈网络}（Position-wise Feed-Forward Network）
\end{enumerate}
\begin{itemize}
  \item 采用\textbf{残差连接}（Residual Connection）：$\text{LayerNorm}(x + \text{Sublayer}(x))$
  \item 所有子层输出维度：\textbf{\%\%INLINEMATH16\%\%}
\end{itemize}

\paragraph{3.1.2 解码器（Decoder）}

\begin{itemize}
  \item 同样由 \textbf{N = 6} 个相同层的堆叠组成
  \item 每层包含三个子层：
\end{itemize}
\begin{enumerate}
  \item \textbf{掩码多头自注意力}（Masked Multi-Head Attention）
  \item \textbf{编码器-解码器注意力}（Encoder-Decoder Attention）
  \item \textbf{逐位置前馈网络}
\end{enumerate}
\begin{itemize}
  \item 同样使用残差连接和层归一化
  \item \textbf{关键机制}：防止位置 $i$ 关注后续位置（保持自回归特性）
\end{itemize}

\subsubsection*{3.2 注意力机制详解}

\paragraph{3.2.1 缩放点积注意力（Scaled Dot-Product Attention）}

这是Transformer核心的注意力函数。

\textbf{输入}：
\begin{itemize}
  \item 查询（Query）：$\mathbf{Q} \in \mathbb{R}^{n \times d_k}$
  \item 键（Key）：$\mathbf{K} \in \mathbb{R}^{m \times d_k}$
  \item 值（Value）：$\mathbf{V} \in \mathbb{R}^{m \times d_v}$
\end{itemize}

\textbf{核心公式}：

\[
\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}}\right)\mathbf{V}
\]

\begin{figure}[htbp]
\centering
\includegraphics[width=0.8\textwidth]{fig2_p4.png}
\end{figure}

\textbf{工作流程}（参见论文\textbf{图2左}）：
\begin{enumerate}
  \item 计算查询与所有键的点积：$\mathbf{Q}\mathbf{K}^T$
  \item 缩放因子 $\frac{1}{\sqrt{d_k}}$ 防止点积过大
  \item 应用softmax获取注意力权重
  \item 加权求和得到输出
\end{enumerate}

\textbf{为什么需要缩放？}

假设 $q$ 和 $k$ 的分量是均值为0、方差为1的独立随机变量，则点积 $q \cdot k = \sum_{i=1}^{d_k} q_i k_i$ 的：
\begin{itemize}
  \item 均值为 0
  \item \textbf{方差为 \%\%INLINEMATH26\%\%}
\end{itemize}

当 $d_k$ 较大时，点积的量级会显著增大，将softmax函数推向\textbf{梯度极小的区域}。缩放因子 $\frac{1}{\sqrt{d_k}}$ 将方差归一化回1。

\textbf{与加性注意力的比较}：
\begin{itemize}
  \item 加性注意力使用前馈网络计算兼容性
  \item 点积注意力实现更简单、速度更快、空间效率更高
  \item 两者理论复杂度相似，但点积注意力可通过高度优化的矩阵乘法实现
\end{itemize}

\paragraph{3.2.2 多头注意力（Multi-Head Attention）}

\textbf{设计动机}：
\begin{itemize}
  \item 不使用单一的高维注意力，而是将查询、键、值\textbf{线性投影}到多个低维子空间
  \item 允许模型在\textbf{不同的表示子空间}中同时关注来自不同位置的信息
  \item 单头注意力的平均操作会抑制这种多样性
\end{itemize}

\textbf{核心公式}：

\[
\text{MultiHead}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{Concat}(\text{head}_1, ..., \text{head}_h)\mathbf{W}^O
\]

其中：

\[
\text{head}_i = \text{Attention}(\mathbf{Q}\mathbf{W}_i^Q, \mathbf{K}\mathbf{W}_i^K, \mathbf{V}\mathbf{W}_i^V)
\]

\textbf{投影矩阵维度}：
\begin{itemize}
  \item $\mathbf{W}_i^Q \in \mathbb{R}^{d_{\text{model}} \times d_k}$
  \item $\mathbf{W}_i^K \in \mathbb{R}^{d_{\text{model}} \times d_k}$
  \item $\mathbf{W}_i^V \in \mathbb{R}^{d_{\text{model}} \times d_v}$
  \item $\mathbf{W}^O \in \mathbb{R}^{hd_v \times d_{\text{model}}}$
\end{itemize}

\textbf{本文的具体设置}：
\begin{itemize}
  \item 头数 $h = 8$
  \item 每个头的维度 $d_k = d_v = d_{\text{model}}/h = 64$
  \item 由于每头维度减小，总计算成本与全维度单头注意力相当
\end{itemize}

\textbf{多头注意力的优势}：
\begin{itemize}
  \item \textbf{并行计算}：各头可同时处理
  \item \textbf{参数效率}：总维度不变，但表达能力增强
  \item \textbf{多样性}：不同头可学习关注不同类型的依赖关系
\end{itemize}

\paragraph{3.2.3 Transformer中注意力的三种应用}

\begin{center}
\begin{tabular}{|l|l|l|l|}
\hline
应用场景 & 查询来源 & 键/值来源 & 作用 \\
\hline
\textbf{编码器自注意力} & 编码器前一层的输出 & 编码器前一层的输出 & 允许编码器关注输入序列的所有位置 \\
\hline
\textbf{解码器自注意力} & 解码器前一层的输出 & 解码器前一层的输出 & 允许解码器关注当前位置及之前的位置 \\
\hline
\textbf{编码器-解码器注意力} & 解码器的上一子层 & 编码器的最终输出 & 允许解码器关注输入序列的相关部分 \\
\hline
\end{tabular}
\end{center}

\textbf{解码器中的掩码机制}：

为保持自回归特性，解码器的自注意力层需要\textbf{防止向左信息流动}。通过将softmax输入中对应非法连接的位置设置为 $-\infty$ 来实现掩码：

\[
\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}} + \mathbf{M}\right)\mathbf{V}
\]

其中 $\mathbf{M}$ 是掩码矩阵，非法位置为 $-\infty$，合法位置为0。

\subsubsection*{3.3 逐位置前馈网络（Position-wise Feed-Forward Networks）}

除注意力子层外，编码器和解码器的每层还包含一个\textbf{全连接前馈网络}，对每个位置独立应用相同的变换：

\[
\text{FFN}(x) = \max(0, x\mathbf{W}_1 + b_1)\mathbf{W}_2 + b_2
\]

或等价地：

\[
\text{FFN}(x) = \text{ReLU}(x\mathbf{W}_1 + b_1)\mathbf{W}_2 + b_2
\]

\textbf{参数配置}：
\begin{itemize}
  \item 输入/输出维度：$d_{\text{model}} = 512$
  \item 内部层维度：$d_{ff} = 2048$
  \item 可视为两个核大小为1的卷积
\end{itemize}

\textbf{作用}：
\begin{itemize}
  \item 为每个位置提供非线性变换能力
  \item 增加模型的表达能力
  \item 计算可完全并行
\end{itemize}

\subsubsection*{3.4 位置编码（Positional Encoding）}

由于Transformer\textbf{不含循环和卷积}，无法自然地利用序列顺序信息。因此，必须注入关于\textbf{令牌相对或绝对位置}的信息。

\paragraph{3.4.1 编码方案}

本文采用\textbf{正弦和余弦函数}的固定位置编码：

\[
PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)
\]

\[
PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)
\]

其中：
\begin{itemize}
  \item $pos$：位置（0, 1, 2, ...）
  \item $i$：维度索引（0, 1, 2, ..., $d_{\text{model}}/2$）
\end{itemize}

\paragraph{3.4.2 设计原理}

对于任意固定偏移量 $k$，位置编码 $PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的\textbf{线性函数}：

\[
PE_{pos+k} = \mathbf{M} \cdot PE_{pos}
\]

其中 $\mathbf{M}$ 是一个旋转矩阵。这使得模型能够\textbf{轻松学习关注相对位置}。

\paragraph{3.4.3 与可学习位置编码的比较}

作者也实验了可学习的位置嵌入，发现两种方法产生了\textbf{几乎相同的结果}。选择正弦版本的原因：
\begin{itemize}
  \item 可外推到\textbf{比训练时更长的序列}
  \item 无需学习额外参数
\end{itemize}

\subsubsection*{3.5 完整算法流程}

\textbf{编码器前向传播}（单层）：

\begin{verbatim}
Input: x (batch_size, seq_len, d_model)
1. x' = LayerNorm(x + MultiHeadSelfAttention(x))
2. output = LayerNorm(x' + FFN(x'))
\end{verbatim}

\textbf{解码器前向传播}（单层）：

\begin{verbatim}
Input: x (decoder input), encoder_output
1. x' = LayerNorm(x + MaskedMultiHeadSelfAttention(x))
2. x'' = LayerNorm(x' + MultiHeadAttention(x', encoder_output, encoder_output))
3. output = LayerNorm(x'' + FFN(x''))
\end{verbatim}

根据论文中的\textbf{表1}，各层类型的复杂度比较：

  \item 自注意力层的\textbf{最大路径长度为常数 \%\%INLINEMATH61\%\%}，优于所有其他方法
\end{itemize}

  \item 每批次约包含 \textbf{25000个源令牌} 和 \textbf{25000个目标令牌}
\end{itemize}

\paragraph{4.3.3 优化器}

使用 \textbf{Adam优化器}，配合学习率预热策略：

\[
lrate = d_{\text{model}}^{-0.5} \cdot \min\left(step\_num^{-0.5}, step\_num \cdot warmup\_steps^{-1.5}\right)
\]

\begin{itemize}
  \item $\beta_1 = 0.9$
  \item $\beta_2 = 0.98$
  \item $\epsilon = 10^{-9}$
  \item $warmup\_steps = 4000$
\end{itemize}

该策略在\textbf{前4000步线性增加学习率}，之后按步数的平方根倒数递减。

\paragraph{4.3.4 正则化}

\begin{enumerate}
  \item \textbf{残差Dropout}：
\end{enumerate}
\begin{itemize}
  \item 对每个子层输出的dropout率 $P_{drop} = 0.1$
  \item 编码器/解码器栈中嵌入与位置编码和的dropout
\end{itemize}

\begin{enumerate}
  \item \textbf{标签平滑}：
\end{enumerate}
\begin{itemize}
  \item $\epsilon_{ls} = 0.1$
  \item 虽增加困惑度，但提升准确率和BLEU分数
\end{itemize}

\paragraph{4.3.5 推理策略}

\begin{itemize}
  \item \textbf{束搜索}（Beam Search）：beam size = 4
  \item \textbf{长度惩罚}：$\alpha = 0.6$
  \item 最大输出长度：输入长度 + 50
\end{itemize}

\subsection*{5. 优缺点评价}

\subsubsection*{5.1 主要优点}

\paragraph{5.1.1 卓越的并行化能力}

Transformer完全基于注意力机制，可以\textbf{在序列的所有位置并行计算}，训练速度比RNN快数个数量级。这使得在合理时间内训练大型模型成为可能。

\paragraph{5.1.2 最优的长距离依赖建模}

\textbf{最大路径长度为 \%\%INLINEMATH74\%\%}，常数级复杂度使模型能够轻松学习任意距离位置间的依赖关系，解决了RNN的梯度消失/爆炸问题。

\paragraph{5.1.3 突破性的翻译质量}

\begin{itemize}
  \item 在WMT 2014英德任务上以 \textbf{28.4 BLEU} 超越所有先前模型
  \item 在WMT 2014英法任务上以 \textbf{41.8 BLEU} 创下单模型记录
  \item 证明了注意力机制本身的质量优势
\end{itemize}

\paragraph{5.1.4 优秀的泛化能力}

成功应用于英语句法分析任务，展示了架构的\textbf{任务无关性}和\textbf{可迁移性}。

\paragraph{5.1.5 可解释性}

注意力权重提供了\textbf{可视化的注意力分布}，使模型的决策过程更加透明，有助于理解模型学到了什么。

\subsubsection*{5.2 主要局限性}

\paragraph{5.2.1 计算复杂度与序列长度的关系}

标准自注意力的复杂度为 $O(n^2 \cdot d)$，当\textbf{序列长度 \%\%INLINEMATH76\%\% 非常大}时（如长文档、音频信号），计算成本显著增加。

\paragraph{5.2.2 位置编码的局限}

使用固定正弦位置编码，虽然可外推，但可能无法完全捕捉复杂的位置关系。相比可学习的位置嵌入，其表达能力可能受限。

\paragraph{5.2.3 训练稳定性}

深层Transformer（6层以上）的训练可能需要仔细的超参数调优，如学习率预热、层归一化位置等。

\subsubsection*{5.3 可能的改进方向}

\paragraph{5.3.1 高效注意力机制}

\begin{itemize}
  \item \textbf{Sparse Attention}：如Longformer、BigBird，通过稀疏模式降低复杂度
  \item \textbf{Linformer}：线性复杂度的注意力近似
  \item \textbf{Reformer}：使用局部敏感哈希的注意力
\end{itemize}

\paragraph{5.3.2 相对位置编码}

\begin{itemize}
  \item \textbf{T5的相对位置编码}：更灵活地处理位置关系
  \item \textbf{Transformer-XL}：段落级别的相对位置编码
\end{itemize}

\paragraph{5.3.3 轻量化模型}

\begin{itemize}
  \item \textbf{DistilBERT}：知识蒸馏压缩
  \item \textbf{MobileBERT}：针对移动端优化
\end{itemize}

\paragraph{5.3.4 多模态扩展}

将Transformer扩展到图像（ViT）、音频、视频等模态。

\subsection*{6. 个人思考与启发}

\subsubsection*{6.1 核心思想启发}

\paragraph{6.1.1 注意力机制的本质力量}

这篇论文最深刻的启示是：\textbf{注意力机制本身足以完成序列转导任务}。之前的工作将注意力视为RNN的辅助工具，而本文证明了注意力可以独立成为模型的核心。

这启示我们：
\begin{itemize}
  \item 重新审视深度学习中"基础组件"的假设
  \item 简洁但正确的设计可以超越复杂的工程优化
\end{itemize}

\paragraph{6.1.2 并行化与序列建模的兼容}

Transformer证明了\textbf{并行计算和序列依赖建模并不矛盾}。通过使用全局注意力，模型可以同时：
\begin{itemize}
  \item 利用GPU/TPU的并行计算能力
  \item 保持对序列中任意位置依赖的建模能力
\end{itemize}

这对其他序列任务的设计具有重要指导意义。

\paragraph{6.1.3 多头注意力的协同效应}

多个注意力头可以学习\textbf{不同类型的依赖关系}：
\begin{itemize}
  \item 某些头专注于局部关系
  \item 某些头专注于长距离依赖
  \item 某些头捕捉语法结构
  \item 某些头捕捉语义关联
\end{itemize}

这表明通过\textbf{模块化设计}，可以让模型的不同组件专注于不同子任务。

\subsubsection*{6.2 可借鉴的技术}

\paragraph{6.2.1 残差连接 + 层归一化}

\[
\text{Output} = \text{LayerNorm}(x + \text{Sublayer}(x))
\]

这一组合已成为现代深度学习模型的标准组件，对训练深层网络至关重要。

\paragraph{6.2.2 学习率预热策略}

\[
lrate = d_{\text{model}}^{-0.5} \cdot \min\left(step\_num^{-0.5}, step\_num \cdot warmup\_steps^{-1.5}\right)
\]

预热阶段帮助模型在早期训练中稳定收敛，尤其适用于大型模型。

\paragraph{6.2.3 标签平滑}

\[
\text{loss} = -\sum_{i} y_i \cdot \left((1 - \epsilon)\log(\hat{y}_i) + \frac{\epsilon}{K}\right)
\]

通过引入适度的不确定性，提升模型的泛化能力。

\subsubsection*{6.3 Follow-up 研究方向}

\paragraph{6.3.1 大规模预训练模型}

\begin{itemize}
  \item \textbf{BERT}：基于Transformer编码器的双向预训练

\end{itemize}

\paragraph{6.3.3 多模态学习}

\begin{itemize}

\subsection*{附录：关键公式汇总}

\textbf{缩放点积注意力}：

\[
\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}}\right)\mathbf{V}
\]

\textbf{多头注意力}：

\[
\text{MultiHead}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{Concat}(\text{head}_1, ..., \text{head}_h)\mathbf{W}^O
\]

\textbf{前馈网络}：

\[
\text{FFN}(x) = \max(0, x\mathbf{W}_1 + b_1)\mathbf{W}_2 + b_2
\]

\textbf{位置编码}：

\[
PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)
\]
\[
PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)
\]

\textbf{学习率调度}：

\[
lrate = d_{\text{model}}^{-0.5} \cdot \min\left(step\_num^{-0.5}, step\_num \cdot warmup\_steps^{-1.5}\right)
\]
